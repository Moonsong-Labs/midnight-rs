# midnight-rs

> [!WARNING]
> This project is under active development and is **not production ready**. APIs may change without notice.

Rust SDK for the [Midnight](https://midnight.network) blockchain. Query contract state, execute circuit logic, build and submit transactions — all from Rust.

## Crates

| Crate | Description |
|---|---|
| `midnight-core` | Meta-crate, re-exports all sub-crates |
| `midnight-provider` | `Provider` trait + `MidnightProvider` (indexer + node RPC) |
| `midnight-contract` | `Contract<P, L>` wrapper, circuit interpreter, transaction builder |
| `midnight-indexer-client` | Typed GraphQL client for the Midnight indexer API |

## Querying contract state

Generate typed bindings from your compiled contract's `contract-info.json`, then read state through a provider:

```rust
use midnight_core::{MidnightProvider, Contract, Provider};

// Generate typed ledger bindings at compile time
mod gateway {
    midnight_core::midnight_bindgen::contract!(
        "path/to/compiled/gateway/compiler/contract-info.json"
    );
}

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let provider = MidnightProvider::new("ws://localhost:9944", "http://localhost:8088")?;

    // Typed contract wrapper — `gateway::Ledger` is generated by the macro
    let contract = Contract::<_, gateway::Ledger>::new("contract_address", &provider);

    // Fetch current state
    let ledger = contract.ledger().await?;
    println!("threshold: {}", ledger.threshold()?);

    // Historical queries
    let old = contract.ledger_at_height(1000).await?;
    let at_block = contract.ledger_at_block_hash("0xabc...").await?;
    let at_tx = contract.ledger_at_tx_hash("0xdef...").await?;

    Ok(())
}
```

The `contract!` macro reads `contract-info.json` and generates a `Ledger` struct with typed accessors for each ledger field. Fields are lazily deserialized — only the fields you access are decoded.

## Calling contract circuits

The SDK can execute circuit logic locally and build transactions for submission to the network. This requires the [extended Compact compiler](https://github.com/RomarQ/compact-compiler-fork) that embeds circuit IR in `contract-info.json`.

### 1. Load the circuit IR

```rust
use compact_codegen::ir::CircuitIrBody;

// Load contract-info.json (with embedded IR)
let info: serde_json::Value = serde_json::from_str(
    &std::fs::read_to_string("compiled/counter/compiler/contract-info.json")?
)?;

// Find a circuit's IR by name
let circuit = info["circuits"].as_array().unwrap()
    .iter()
    .find(|c| c["name"].as_str() == Some("increment"))
    .unwrap();
let ir: CircuitIrBody = serde_json::from_value(circuit["ir"].clone())?;

// Load helper functions (pure functions called by circuits)
let helpers: Vec<compact_codegen::ir::HelperDef> = info["helpers"]
    .as_array()
    .map(|arr| arr.iter().filter_map(|h| serde_json::from_value(h.clone()).ok()).collect())
    .unwrap_or_default();
```

### 2. Execute locally

Execute circuit logic against a contract state without building a transaction. Useful for simulation, validation, and reading computed values.

```rust
use midnight_contract::interpreter::{self, Value, WitnessProvider};
use midnight_bindgen::{ContractState, InMemoryDB, StateValue, StorageHashMap,
                       ContractMaintenanceAuthority};

// Build or fetch the contract state
let state = ContractState::new(
    StateValue::Array(vec![StateValue::from(0u64)].into()),
    StorageHashMap::new(),
    ContractMaintenanceAuthority::default(),
);

// Execute with no arguments, no witnesses, no helpers
let result = interpreter::execute(&ir, &state)?;
println!("new state root: {:?}", result.state.data.get_ref());

// Execute with arguments and witnesses
let result = interpreter::execute_with(
    &ir,
    &state,
    &[("amount", Value::Integer(100))],   // circuit arguments
    &MyWitnessProvider,                     // private state callbacks
    &helpers,                               // helper functions from contract-info.json
)?;
```

### 3. Provide witnesses (private state)

Circuits that access private state call witness functions. Implement the `WitnessProvider` trait to supply these values:

```rust
use midnight_contract::interpreter::{InterpreterError, Value, WitnessProvider};

struct MyWitness {
    secret_key: [u8; 32],
}

impl WitnessProvider for MyWitness {
    fn call_witness(&self, name: &str, _args: &[Value]) -> Result<Value, InterpreterError> {
        match name {
            "private$secret_key" => Ok(Value::AlignedValue(
                midnight_bindgen::AlignedValue::from(self.secret_key)
            )),
            _ => Err(InterpreterError::Witness(format!("unknown witness: {name}"))),
        }
    }
}
```

### 4. Build and submit transactions

Build a full transaction from circuit execution, generate ZK proofs, and submit to the network:

```rust
use midnight_contract::call;
use midnight_coin_structure::contract::ContractAddress;

let address = ContractAddress(midnight_base_crypto::hash::HashOutput([0xAA; 32]));

// Step 1: Build unproven transaction (executes circuit, builds transcripts)
let unproven = call::build_unproven_call_tx(
    &ir, &state, "increment", address, "mainnet"
)?;
println!("unproven TX: {} bytes", unproven.tx_bytes.len());
println!("new counter: {:?}", unproven.new_state.data.get_ref());

// Step 2: Generate ZK proofs (requires compiler output with keys/ and zkir/)
let proven_bytes = call::prove_and_seal(&unproven, "compiled/counter").await?;

// Step 3: Submit to node via subxt
use subxt::{OnlineClient, SubstrateConfig};

let client = OnlineClient::<SubstrateConfig>::from_insecure_url("ws://localhost:9944").await?;
let tx = subxt::dynamic::tx(
    "Midnight",
    "send_mn_transaction",
    vec![subxt::dynamic::Value::from_bytes(&proven_bytes)],
);
let unsigned = client.tx().await?.create_unsigned(&tx)?;
unsigned.submit().await?;
```

### 5. Deploy contracts

```rust
use midnight_contract::call;
use midnight_onchain_runtime::state::{ContractOperation, EntryPointBuf};

// Register circuit entry points in the initial state
let entry_point: EntryPointBuf = b"increment"[..].into();
let mut operations = StorageHashMap::new();
operations = operations.insert(entry_point, ContractOperation::new(None));

let initial_state = ContractState::new(
    StateValue::Array(vec![StateValue::from(0u64)].into()),
    operations,
    ContractMaintenanceAuthority::default(),
);

// Build deploy transaction (uses mock_prove — no ZK keys needed)
let (address, tx_bytes) = call::build_deploy_tx(&initial_state, "mainnet")?;
println!("contract address: {}", hex::encode(address.0 .0));

// Submit tx_bytes to node (same as step 3 above)
```

## Supported contract features

The circuit interpreter supports the full Compact language:

- Arithmetic: `add`, `sub`, `mul`
- Comparison: `eq`, `neq`, `lt`, `le`, `gt`, `ge`
- Boolean: `and`, `or`, `not`
- Control flow: `if`, `if-else`, `if-expr`, `assert`
- Ledger operations: `idx`, `ins`, `addi`, `push`, `popeq`, `member`, `rem`, `root`, `eq`, `ckpt`, `dup`, `noop`
- Function calls: pure helpers, witness callbacks, runtime builtins
- Runtime builtins: `persistentHash`, `leafHash`, `ecMulGenerator`, `pad`, `disclose`
- Type conversions: `cast`, `default`, `new`
