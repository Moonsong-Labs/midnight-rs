# AlignedValue and state navigation

> Internal reference for SDK developers. Users of midnight-rs interact with typed accessors generated by `contract!`, not raw `AlignedValue`s.

## What is AlignedValue?

`AlignedValue` is the universal value type in Midnight's runtime. Every piece of data stored in contract state, passed to circuits, or used as a map key is represented as an `AlignedValue`.

```rust
pub struct AlignedValue {
    pub value: Value,           // the data (sequence of ValueAtoms)
    pub alignment: Alignment,   // describes the data's type/shape
}
```

### Value

A sequence of `ValueAtom`s (byte vectors). Simple types produce one atom, compound types produce multiple:

- `u64(42)` → one atom: `[0x2a]`
- `bool(true)` → one atom: `[0x01]`
- `{ a: u64, b: u64 }` → two atoms: `[bytes_of_a, bytes_of_b]`

### Alignment

Describes how to interpret the atoms:

| Rust type | Alignment | Compact type |
|-----------|-----------|-------------|
| `u8` | `Bytes(1)` | `Uint<8>` |
| `u16` | `Bytes(2)` | `Uint<16>` |
| `u32` | `Bytes(4)` | `Uint<32>` |
| `u64` | `Bytes(8)` | `Uint<64>` / `Counter` |
| `u128` | `Bytes(16)` | `Uint<128>` |
| `bool` | `Bytes(1)` | `Boolean` |
| `[u8; N]` | `Bytes(N)` | `Bytes<N>` |
| `Fr` | `Field` | `Field` |
| tuples | concatenated | structs |

## Contract state tree

A Compact contract's state is nested `StateValue` nodes:

```
StateValue::Array                        ← top level (ledger fields by index)
  ├── [0] StateValue::Cell(AlignedValue) ← threshold: Uint<8>
  ├── [1] StateValue::Map(HashMap)       ← egress_jobs: Map<Field, EgressJob>
  │         ├── Fr(1) → Cell(EgressJob)
  │         └── Fr(2) → Cell(EgressJob)
  └── [2] StateValue::Cell(AlignedValue) ← job_count: Counter
```

## How the `contract!` macro uses this

The generated `Ledger` struct navigates this tree:

```rust
// Generated code (simplified):
pub fn threshold(&self) -> Result<u8, StateError> {
    let sv = get_field(self.state.data.get_ref(), FIELD_THRESHOLD)?;
    let av = cell_value(sv)?;
    <u8>::try_from(&*av.value).map_err(StateError::Conversion)
}
```

`get_field(root, index)` does `Array[index]`, `cell_value(sv)` extracts the `AlignedValue` from a `Cell`, and `TryFrom<&ValueSlice>` converts to the typed Rust value.

## How the interpreter uses this

The circuit interpreter's `Idx` ledger op navigates the state tree using `AlignedValue` keys:

| StateValue variant | Key interpretation |
|---|---|
| `Array` | Convert to `u8` index |
| `Map` | HashMap key lookup |
| `BoundedMerkleTree` | Convert to `u64` leaf position |

The `path_value_to_aligned()` function in the interpreter converts IR path entries to `AlignedValue` keys, respecting the declared type (Uint, Field, Boolean, etc.).

## Lazy query paths

The `midnight_queryContractState` RPC uses serialized `AlignedValue`s as path steps:

```json
{
  "path": ["4001", "0141"]
}
```

Where `"4001"` is hex-encoded `AlignedValue::from(0u8)` (field index 0) and `"0141"` is `AlignedValue::from(Fr(1))` (map key).

The generated `LedgerQuery` struct (lazy accessor) builds these paths automatically via `index_to_query_key()` and `value_to_query_key()`.
